--- layout: presentation --- # Simple Storage Service (s3) Contact: [rleone@scaleway.com](mailto:rleone@scaleway.com) --- # Problem it solves - How can I store blob of data? - How can I retrieve them? --- # Lifecycle - Create a bucket (name is unique worldwide) - Add an object to the bucket - View an object - Move an object - Delete an object --- # Common use-cases - Backup and Storage – Provide data backup and storage services for others. - Application Hosting – Provide services that deploy, install, and manage web applications. - Media Hosting – Build a redundant, scalable, and highly available infrastructure that hosts video, photo, or music uploads and downloads. - Software Delivery – Host your software applications that customers can download. - Write once, Read many --- # Concepts --- # Buckets - Container for objects stored in S3 - Organize namespace - Unit of aggregation - Unique name worldwide --- # Objects - Fundamental entities stored in S3 - Unique within a bucket - Have an access policy - Can have a public URL - Bucket Name & Object Name is globally unique --- # Regions - Geographical region in which an object is stored --- # Tools - [Cyberduck](https://www.scaleway.com/docs/store-object-with-cyberduck/) - [s3cmd](https://www.scaleway.com/docs/object-storage-with-s3cmd/) (Config [example](code/s3cmd.ini)) - [Web Console](https://cloud.scaleway.com/#/buckets) - [REST API](https://www.scaleway.com/docs/s3-object-storage-api/) - [SDK](https://aws.amazon.com/tools/) --- # Exercice - Upload this website to a bucket with those tools - Check that this website is available online - Beware of private/public access
